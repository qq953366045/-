note3

3.1 初识Hexo
3.1.1 创建博客文件夹：
hexo init blog
此处blog为文件夹名字，可任意取名
3.1.2 安装发布工具：（必须在博客文件夹目录下安装！）
npm install hexo-deployer-git –save
3.1.3 修改配置文件
找到_config.yml文件，打开输入：
theme:landscape
deploy:
type:git
repository:你的GitHub仓库地址 git@github.com:qq953366045/qq953366045.github.io.git
branch:master
3.1.4 Hexo提交
1：hexo clean：清除缓存，能让新发布的博客快速生效
2：hexo generate：生成新的静态文件，可以简写为hexo g
3：hexo deploy：把生成的文件部署到博客上，可以简写为hexo d

3.2 Hexo的应用
3.2.1 创建博客：hexo new 博客名
该命令会创建一个新的md文件存放在source/_posts文件夹内
3.2.2 发布博客：hexo提交
3.2.3 更换主题：（更换landscape为next）
在博客跟目录中下载next新主题：
git clone https://github.com/iissnan/hexo-theme-next themes/next
这里https://github.com/iissnan/hexo-theme-next是该主题的仓库地址，后面的themes/next表示下载到themes文件夹下
更改文件配置：打开_config.yml，将theme： landscape改为theme： next（：后有个空格）
发布更改：提交hexo

3.3 Git分支&Git ignore
3.3.1 创建分支：git branch 分支名
切换分支：git checkout dev（创建的分支名为dev）
也可以直接用git checkout -b dev（相当于创建了dev分支并切换到它）
3.3.2 查看gitignore文件
.DS_Store
Thumbs.db
db.json
.lognode_modules/
public/
.deploy/
表示提交时这些不会被提交到线上仓库，如果没有public/得自己添加一行

3.4 本地电脑部署博客
首先安装git
本地配置ssh并绑定至GitHub
先生成本地SSH：
ssh-keygen -t rsa -b 4096 -C “953366045@qq.com“
之后执行cat ~/.ssh/id_rsa.pub将获取的内容复制到GitHub中就行了
然后安装node.js，之后本地安装Hexo：
npm install -g hexo-cli
之后将dev分支的代码git clone到本地
git clone git@github.com:qq953366045/qq953366045.github.io.git dev

note2

2.1.1 查看本地SSH
cat ~/.ssh/id_rsa.pub
2.1.2 生成本地SSH
ssh-keygen -t rsa -b 4096 -C “your_email@example.com“ 此处邮箱替换为注册GitHub的邮箱
之后重新执行cat ~/.ssh/id_rsa.pub将获取的内容复制到GitHub中就行了。
2.2.1 配置全局变量：
1）git config –global user.name “沈高昂”
告诉git你的名字
2）git config –global user.email “953366045@qq.com“
告诉git你的邮箱
2.2.2 获取仓库地址
git clone git@github.com:qq953366045/qq953366045.github.io.git
2.2.3 常见Linux命令
1.查看当前目录下的文件：ls
2.查看当前目录下的文件，包括隐藏文件：ls -a
3.创建文件：touch 文件名
4.创建文件夹：mkdir 文件夹名
5.进入某文件夹：cd 文件目录
6：删除文件：rm -rf 文件名
2.3.1 提交到GitHub上
步骤1：git add：可以用git add -A或者git add . 2个命令效果一样
步骤2：git commit：git commit -m “本次提交的修改的备注”
也可以直接用git commit -am “本次提交的修改的备注”来合并前2个步骤。
步骤3:git push:
第一次提交到本分支：git push origin master
origin是远程仓库的默认名称，master是我们的分支名称（主分支）
第二到n次提交到本分支可以直接用git push
提交到其他分支：git push origin b（会提交到b分支）
2.3.2 抓取远程仓库内容
git pull
2.4 绑定远程仓库
2.4.1 初始化git仓库：用git init将一个文件夹初始化为一个git仓库
2.4.2 绑定本地仓库：
git remote -v：查看当前git仓库绑定的远程仓库，显示空白即无绑定。
git remote add origin 仓库地址 （git@github.com:qq953366045/youkeda.git）
绑定成功后用git remote -v可以查看是否绑定成功。
绑定错误可以用git remote remove origin来移除绑定

note1

1.1：使用#表示1-6级标题
比如二级标题：##这是一个二级标题
1.2：加粗体/加斜体：
加粗的部分 表示加粗；加斜的部分 表示加斜
1.3：插入图片：
！[]（图片的url）
1.4：列表：
无序列表：句首加*
有序列表：句首加1.
*和1.后面都有空格，无序列表是句首是点的效果，有序列表是句首是1，2，3的效果
1.5：代码块：
、行内代码、
、、、行间代码、、、
