2.1.1 查看本地SSH
cat ~/.ssh/id_rsa.pub
2.1.2 生成本地SSH
ssh-keygen -t rsa -b 4096 -C “your_email@example.com“ 此处邮箱替换为注册GitHub的邮箱
之后重新执行cat ~/.ssh/id_rsa.pub将获取的内容复制到GitHub中就行了。
2.2.1 配置全局变量：
1）git config –global user.name “沈高昂”
告诉git你的名字
2）git config –global user.email “953366045@qq.com“
告诉git你的邮箱
2.2.2 获取仓库地址
git clone git@github.com:qq953366045/qq953366045.github.io.git
2.2.3 常见Linux命令
1.查看当前目录下的文件：ls
2.查看当前目录下的文件，包括隐藏文件：ls -a
3.创建文件：touch 文件名
4.创建文件夹：mkdir 文件夹名
5.进入某文件夹：cd 文件目录
6：删除文件：rm -rf 文件名
2.3.1 提交到GitHub上
步骤1：git add：可以用git add -A或者git add . 2个命令效果一样
步骤2：git commit：git commit -m “本次提交的修改的备注”
也可以直接用git commit -am “本次提交的修改的备注”来合并前2个步骤。
步骤3:git push:
第一次提交到本分支：git push origin master
origin是远程仓库的默认名称，master是我们的分支名称（主分支）
第二到n次提交到本分支可以直接用git push
提交到其他分支：git push origin b（会提交到b分支）
2.3.2 抓取远程仓库内容
git pull
2.4 绑定远程仓库
2.4.1 初始化git仓库：用git init将一个文件夹初始化为一个git仓库
2.4.2 绑定本地仓库：
git remote -v：查看当前git仓库绑定的远程仓库，显示空白即无绑定。
git remote add origin 仓库地址 （git@github.com:qq953366045/youkeda.git）
绑定成功后用git remote -v可以查看是否绑定成功。
绑定错误可以用git remote remove origin来移除绑定
