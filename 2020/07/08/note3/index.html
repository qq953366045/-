3.1 初识Hexo
3.1.1 创建博客文件夹：
hexo init blog
此处blog为文件夹名字，可任意取名
3.1.2 安装发布工具：（必须在博客文件夹目录下安装！）
npm install hexo-deployer-git –save
3.1.3 修改配置文件
找到_config.yml文件，打开输入：
theme:landscape
deploy:
type:git
repository:你的GitHub仓库地址 git@github.com:qq953366045/qq953366045.github.io.git
branch:master
3.1.4 Hexo提交
1：hexo clean：清除缓存，能让新发布的博客快速生效
2：hexo generate：生成新的静态文件，可以简写为hexo g
3：hexo deploy：把生成的文件部署到博客上，可以简写为hexo d

3.2 Hexo的应用
3.2.1 创建博客：hexo new 博客名
该命令会创建一个新的md文件存放在source/_posts文件夹内
3.2.2 发布博客：hexo提交
3.2.3 更换主题：（更换landscape为next）
在博客跟目录中下载next新主题：
git clone https://github.com/iissnan/hexo-theme-next themes/next
这里https://github.com/iissnan/hexo-theme-next是该主题的仓库地址，后面的themes/next表示下载到themes文件夹下
更改文件配置：打开_config.yml，将theme： landscape改为theme： next（：后有个空格）
发布更改：提交hexo

3.3 Git分支&Git ignore
3.3.1 创建分支：git branch 分支名
切换分支：git checkout dev（创建的分支名为dev）
也可以直接用git checkout -b dev（相当于创建了dev分支并切换到它）
3.3.2 查看gitignore文件
.DS_Store
Thumbs.db
db.json
.lognode_modules/
public/
.deploy/
表示提交时这些不会被提交到线上仓库，如果没有public/得自己添加一行

3.4 本地电脑部署博客
首先安装git
本地配置ssh并绑定至GitHub
先生成本地SSH：
ssh-keygen -t rsa -b 4096 -C “953366045@qq.com“
之后执行cat ~/.ssh/id_rsa.pub将获取的内容复制到GitHub中就行了
然后安装node.js，之后本地安装Hexo：
npm install -g hexo-cli
之后将dev分支的代码git clone到本地
git clone git@github.com:qq953366045/qq953366045.github.io.git dev
